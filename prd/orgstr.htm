Организационная диаграмма

Решение "Организационная диаграмма" было введено в состав Visio где-то в районе 2000 года и с тех пор считается одним из базовых, наиболее востребованных решений. Построение организационных диаграмм рассматривается почти в каждой книжке по Visio.

Базовые возможности решения за это время изменились не сильно. Существует несколько шейпов для разных ролей и программные дополнения, ускоряющие работу. Шейпы умеют автоматически присоединяться к существующим при "набрасывании" их на рисунок. Элементы схемы легко выравниваются по указанному правилу. Всегда можно автоматически перерисовать (переразместить) схему.

Помимо специальной панели с командами размещения-выравнивания шейпов есть визард (ассистент) для импорта данных из внешних источников и автоматического построения организационной диаграммы по этим данным. В качестве источников данных можно использовать:

  • Microsoft Exchange Server;
  • файлы Excel;
  • ODBC-совместимые источники.

То есть круг типов источников довольно широк. Привязка полей источника к отображаемым данным производится тем же визардом. Настраивать и отображать можно до пяти полей данных.

Существует также отдельная утилита, с помощью которой можно создать файл организационной диаграммы, не открывая Visio. Настройка производится с помощью параметров, указываемых при запуске исполняемого файла утилиты. Набор параметров примерно такой же, как и при использовании визарда.

Наращивание функциональности в последних версиях Visio (2010 и 2013) шло за счет "раскрашивания". Появилась возможность использовать стили, меняющие цветовую гамму и даже форму шейпов. То есть уже нарисованная диаграмма легким движением руки может быть преобразована к совершенно другому виду. В Visio 2013 появилась еще и возможность импорта фотографий сотрудников из Microsoft Exchange Server. Ian Mikutel хорошо представил возможности Visio 2013 в статье "Visualize your organization like never before in the new Visio". На русском языке эта статья представлена в блоге technet "Беспрецедентно эффективная визуализация данных вашей организации".

Ограничения штатного решения

Но нам ведь важно знать и о подводных камнях - чего не могут штатные средства, в каких случаях может потребоваться разработка или покупка других решений.

1. В конкретном применении может понадобиться вывести на схему более подробные данные о сотрудниках, для которых просто не хватит полей в штатных шейпах. Либо длина выводимых данных не уложится в отведенные им поля.

2. Штатное решение настроено на древовидное построение схемы. Шейпы можно выбрать различные, но вид дерева будет всегда примерно одинаковый. Штатным решением нельзя построить схему с вложенными элементами, как например, на оргсхеме Хаббарда.

3. Кому-то может также не понравиться алгоритм нарезки схемы при размещении ее на нескольких листах.

4. Могут возникнуть трудности при попытке импорта фотографий из других источников данных, не Microsoft Exchange Server.

5. Наконец, если создается более широкое приложение, в котором функци я работы с оргдиаграммой является только одним из входящих элементов, то может понадобиться интеграция этой функции в приложение.

Вот в таких или похожих случаях приходится своими силами или с помощью сторонней организации строить специализированное решение, выполняющее нужную задачу. Причем большую часть работы и в специализированном решении выполняет Visio. Разрабатывается обычно только небольшая надстройка, приспосабливающая Visio к нужной задаче.
